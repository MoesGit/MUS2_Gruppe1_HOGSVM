HOG_SVM-detection

splitting_images

1.) Resizes images and crops them into NxM images.
2.) Overlap of cropped images is possible. Cropped images are manually reviewed and evaluated if dooor knob is in picture or not.
3.) This evaluated cropped pictures are saved into to different folders:
"../Data/images/split_images/positiv"
"../Data/images/split_images/negativ"

HOG

1.) Pushes the positive and negative cropped images in acording image vectors.
2.) Computes the HOG descriptor of positive and negative images an writes them in two CVS files:
"../Data/positiv"
"../Data/negativ"

SVM

1.) The main function opens the CSV files with the hog descriptors for cropped images, split up in two files: negative and positive detection.
2.) The data from the csv files is rearranged in a way, the SVM function can compute it.
3.) The svm is trained with the data from the csv files.
4.) The support vector machine is tested with a test image and a test video.
5.) Predicts door handle in the cropped image and draws a blue square over it.
6.) Calculates a red rectangle from the minimum and maximum coordinates of the blue rectangles, which includes all recognized cropped images.
7.) Red rectangle is the result of the SVM modell.
